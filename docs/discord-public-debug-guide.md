# RHODES OBS COMMANDER3373 公開デバッグ版

統合戦略の状態をADBとMAA-OCRで取得し、OBSへ表示するテスト版です。

## 起動

1. 配布ZIPをすべて展開します。ZIP内から直接起動しないでください。
2. 展開先の `RhodesSuki.exe` を実行します。
3. Windows SmartScreenが出た場合は、発行元と配布元を確認してから「詳細情報」「実行」を選びます。
4. 保存先を聞かれた場合は「実行ファイル側」を選ぶと、設定と報告データが配布フォルダ内にまとまります。

## ADB接続と取得

1. エミュレーターを起動し、ゲームを1280x720の16:9横画面にします。
2. 左の「ランタイム」で「自動検出」を押します。
3. 候補が複数ある場合は、使用中のエミュレーターとserialを選びます。
4. 「接続・撮影」を押し、ゲーム画面が取得できることを確認します。
5. ゲームを探索中のマップ画面に戻し、「ラン」の取得ボタン、または上部の「ADB取得・認識・反映」を実行します。
6. オペレーターと秘宝は「選択」の「オペ認識」「秘宝認識」から取得します。処理中はゲームを操作しないでください。

誤認識や取りこぼしは「ラン」「特殊値」「選択」から手動修正できます。公開デバッグ版の自動認識はIS#5サルカズを優先しています。

## OBS表示

1. 「出力」で「Node.js導入」を押します。インストーラーと管理者権限は不要です。
2. 配信サーバーを起動します。
3. カスタムOverlay URLをコピーし、OBSの「ソース」「ブラウザ」のURLへ貼り付けます。
4. 統合Overlayは幅1920、高さ1080が基準です。個別部品は表示されたURLと推奨サイズを使います。

ADBと手動入力だけならNode.jsは不要です。

## 不具合報告

上部の「報告ZIP」を押し、生成されたZIPをDiscordへ添付してください。

本文には「使用エミュレーター」「実行した取得項目」「期待した結果」「実際の結果」を書いてください。報告ZIPは実行フォルダ内の `RHODES OBS COMMANDER3373 Debug Logs/Bug Reports` に生成されます。

Android Backキーは使用しません。タップとスワイプは指定範囲内でランダム化されています。
